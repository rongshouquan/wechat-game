## 一、建筑升级成本曲线（纯星矿）

**公式**：`升级成本(L→L+1) = round( BASE × L^1.3 )`，`BASE = 120`（v0.1）。逐级上升、与星舰升级同指数(1.3)保手感一致。

| 升级 | L→L+1 | 星矿成本 |
|---|---|---|
| 1→2 | 1 | 120 |
| 2→3 | 2 | 295 |
| 3→4 | 3 | 500 |
| 4→5 | 4 | 727 |
| 5→6 | 5 | 972 |
| 6→7 | 6 | 1,232 |
| 7→8 | 7 | 1,506 |
| 8→9 | 8 | 1,792 |
| 9→10 | 9 | 2,088 |
| **单栋 1→10 合计** | | **≈ 9,232** |

- **8 栋全满 ≈ 7.4 万星矿**（不会同时满，按需升）。前期典型：D7 把船坞升到 3 级（够星舰 15 级）只花 120+295=**415 星矿**，很轻。
- **工人**减免的就是这条星矿成本（-X%，见 6b-4）。
- 可选（待 Ron 定）：给"核心养成建筑"（船坞 / 驾驶员训练舱）加 **×1.2~1.3 系数**让它更有分量；v0.1 默认 8 栋同一条曲线。

## 二、各级效果（8 栋）

分三段沿用现有 lv1 / lv2-5 / lv6-10。**粗体 = 真实数值；其余为功能解锁 / 外观**。

### 1. 船坞 dock（星舰养成中枢）
- **星舰等级上限 = 船坞等级 × 5**（船坞1→星舰可到5级；船坞10→可到50级）。
- lv1 解锁星舰升级；lv6 解锁星舰改装(mod)；2-5/6-10 外观升级。
- 对齐 B1：D7 船坞3→星舰15 / D14 船坞5→25 / D21 船坞7→35 / D28 船坞9→45，与 B1 §4 主力星舰等级线完全吻合。

### 2. 驾驶员训练舱 pilot_training_bay（驾驶员养成中枢）
- **驾驶员等级上限 = 训练舱等级 × 5**（与船坞对称）。
- lv1 解锁驾驶员升级 / 升星 / 看详情；升星(1→5★)消耗驾驶员碎片(不被建筑等级卡，建筑只开功能)。

### 3. 居住舱 habitat（离线 + 人口）
- **离线存储上限**：lv1 = **36 小时(1.5 天)** → 每级 +1.3h → lv10 ≈ **48 小时(2 天)**。守 §12 红线：封顶不超 2 天，"少玩 3-4 天仍明显落后"。
- **基础离线产率加成**：+**2%/级**（lv10 = +18%）。居民的产率/存储加成在 6b-4 叠加在此之上。
- lv1 解锁居民人口面板；高级解锁更热闹外观。

### 4. 星港补给站 supply_station（招募/抽卡建筑）⚠️待 Ron 定
- lv1 解锁三池抽卡（招募/整备/专属）。
- **升级给什么福利属"抽卡/广告策略"= Ron 的域**——v0.1 仅占位"便利向"（如高等级解锁批量抽 UI / 专属池进度显示）。**不得 pay-gate 抽卡质量**（风控）。抽卡出率/保底见 §10.1，不在此。
- 建议你定：要不要给"每日免费招募 1 次"放在某等级解锁（这是慷慨度杠杆，你拍）。

### 5. 打捞港 salvage_port（打捞队 + 信标）
- **打捞队数量**：lv1=**1 队** → lv4=**2 队** → lv7=**3 队**（v1.0 §7「1→3」）；lv8-10 提升信标发现品质 + 外观。
- 信标掉率/品质精确值见 §10.2，不在此。

### 6. 商人小站 merchant_station（商店/回收/刷新）
- lv1 解锁商店 + 回收。
- **商品槽位**：lv1=**2 个** → 每 2 级 +1 → lv10=**6 个**；**每日免费刷新次数**：lv1=**1** → lv6=**2** → lv10=**3**。
- 回收价 = recycle_param / §10.3，不在此（回收换星贝，折损率第二块定）。

### 7. 研究塔 research_tower（全队小加成 · minor_non_gate）
- **全队主属性（攻击+生命）+1%/级 → lv10 = +10%**。这是建筑里仅有的两处"沾战力"之一，刻意做小（核心战力 D1→D28 涨 ×100+，+10% 只是锦上添花、非过墙必需）。
- lv1 解锁基地科技面板；高级解锁研究员便利。

### 8. 星核展厅 core_gallery（收藏 + 轻量加成 · minor_non_gate）
- **全队主属性 +0.5%/级 → lv10 = +5%**（比研究塔更小，因它是后期可选收藏建筑）。
- **收藏里程碑小加成**：每展示满 N 颗星核额外 +小%（v0.1：每 3 颗 +1%，上限 +5%）。
- lv1 解锁展厅（拿到首核后开，约 D3）。

> 研究塔 + 星核展厅的 % 都可由你上调/下调；我按"宁小勿大"取，保证建筑不变成过墙拐杖。

## 三、三条养成线整理（手感对齐）

| 养成线 | 花什么 | 升级成本公式 | 等级范围/上限 | 每级给什么 | 上限受谁卡 |
|---|---|---|---|---|---|
| **建筑** | **星矿** | `120 × L^1.3`（本草案） | 1-10 | 功能/产出/便利 + 研究塔展厅小加成 | 固定 10 级 |
| **星舰** | **合金** | `50 × L^1.3`（B1 §2） | 1-50 | 战力（基础属性）↑ | **船坞等级 ×5** |
| **驾驶员** | **驾驶记录** | `40 × L^1.2`（B1 §2） | 1-50 | 战力（驾驶系数/属性）↑ | **训练舱等级 ×5** |

- **三线各花各的货币**（星矿/合金/驾驶记录），互不抢资源、各自产出渠道独立 → 不会"升了船就没钱升楼"。
- **手感一致**：星舰指数 1.3（主力、最贵）> 驾驶员 1.2（副、略便宜）；建筑同 1.3 但只到 10 级、是"间歇大投资"。
- **闭环**：建筑（船坞/训练舱）抬高星舰/驾驶员的等级上限 → 你升楼 → 解锁更高级 → 战力破墙 → 推图给更多星矿/合金/驾驶记录 → 再升。环环相扣。

## 四、交叉核对之前数值设计（Ron 要求"顺便检验"）—— 发现 2 个要你拍的点

1. ⚠️ **星舰战力有"两套不兼容的模型"，建议统一**：
   - **B1 §1（我做的）**：战力靠**阶级**驱动——阶级基值 C100/B160/A250/S380/SS550（每阶约 ×1.5），等级只 +8%/级（小头）。
   - **引擎配置 `growth_band_param`（Codex 03-04 v0.2 旧设计）**：战力靠**等级**驱动——单条 band_linear，星舰 lv1→40 战力 120→2200（×18，等级是大头），没有阶级乘区。
   - 两者哲学相反（阶级驱动 vs 等级驱动），数字也对不上。**需你拍：以哪套为准**（我建议以 B1 的"阶级驱动"为准——更契合"抽到高阶本体=大跳"的收集爽点；旧 growth_band 要么改造对齐、要么明确它只是别的用途）。这不影响建筑（建筑不进战力公式），但影响星舰/驾驶员最终落地，**早定早省返工**。

2. ❓ **`growth_band_param` 的 "power" 口径不明**：它是"单舰总战力"还是"某个属性分量"？与 B1 的"星舰基础"不是一回事（B1 基础更小、还要 ×驾驶员 +插件 +核）。**需确认口径**，否则两源混用会算错战力。

> 这两点都属"星舰/驾驶员"线，是之前别的对话里做的；我只负责标出来给你拍，不擅自改它们。
