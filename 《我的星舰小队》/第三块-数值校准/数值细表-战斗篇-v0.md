## §0 引擎能力快照（盘点的标尺 · 实读 2026-07-07）

- **积木 4 类**：modifier（flat/pct/set 改 6 基础属性）· affix（8 词条累加）· behavior（覆盖 targetingTag）· action（换普攻/大招/星核槽效果）· trigger（收集给运行时）。
- **词条 8 个**：critRate / critDmg / shieldBreak / skillHaste / healPower / controlResist / dmgVsSwarm / dmgVsBoss。
- **触发 7 种**：battle_start（一次）/ cd（循环·开局即放·吃 skillHaste）/ hp_below（一次）/ on_kill（每有击杀的 tick 可重复）/ on_hit（受击 tick 可重复）/ ally_down（一次）/ passive（装配期语义）。
- **效果类型**：伤害 5 型（basic/clear_barrage/line_pierce/backline_strike/burst_nuke）· 护盾 2 · 治疗 1 · 召唤 2 · 状态 7；伤害型可附带 stateTag（打+挂状态一发完成）。
- **状态 8 个**：shield / shield_break / mark / vulnerable / short_circuit / stun / summon（仅标签）/ berserk（硬编码 ×1.25 攻 ×0.8 间隔）。同名不叠层只刷新时长；硬控（short_circuit/stun）挡普攻+触发。
- **目标标签 8 个**：nearest_random_tie（默认·mark 优先）/ single_target / all_enemies / backline_first / column_line / marked_first / self_team / lowest_hp_ally。范围限制仅普攻生效。
- **公式**：伤害 = 攻 × 倍率 × 100/(100+防)，易伤 ×1.25，×(1+dmgVs*)，暴击掷 RNG（critRate>0 才掷）；护盾量 = max(目标最大血×0.2, 施者攻×倍率)，非叠加取大；治疗 = 施者攻×倍率×(1+healPower)。
- **召唤**：预算=单次 maxTargets + 空格双约束；**无寿命、无随源消亡、无同源场上上限**（真源 §0 要求三者皆有——见缺口）。
- **Boss 阶段**：battle_start / hp_pct_below / time_elapsed 三型，每场一次，放效果+召唤（带 summonCountCap）。
- **既有受控并行加法先例**：回廊 inlineEnemyUnits + timeLimitSecOverride（缺省不传=零行为变化）——本盘点"小旋钮"分级全部对标此先例。
- **遗留字段**：passiveEnergyPerSec 全程无消费点（能量制取消的遗迹），校准时按 0 处理、不赋含义。
- **射程口径**：GDD §4 首发无限射程（射程预留不启用），但现占位行 rangeCells=1~7 且 dist() 对"友方目标"也按敌我公式算距（值无几何意义但确定）——第一段落数时全舰 rangeCells 统一取 99（=事实无限·含支援舰打友军），进常量表；引擎零改动。

**分级图例**：
- ✅ 直通——现有积木直接表达，进第一段填数；
- ≈ 近似直通——用现有积木近似落地、偏差已注明，填数可先行（终态待机制批对齐）；
- 🔧 小旋钮——受控并行加法可补（新枚举值/新字段/新词条，缺省缺席=引擎逐字节不变）；标注成本 小/中；
- ❌ 新机制类型——需新 tick 结算语义/状态生命周期/结算规则，**交回总控**（按任务单授权链不在⑥内做）。
