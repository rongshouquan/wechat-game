# 附录A美术风格细节

> 用途：作为《我的星舰小队》后续美术效果图、透明底素材、背景图、UI组件、图标、特效参考图的统一画风要求。
> 本文件可直接作为 Codex 出图指令中的“项目画风要求”片段引用。

## 项目画风要求

- 微信小游戏，竖屏手机比例。
- 总体画风明亮、温馨、暖色为主，有 painted 成品质感。
- 主题是轻科幻 + Q萌拟人 + 幽默热闹。
- Q版动漫可爱，但不能幼稚；驾驶员要在可爱的基础上更高冷酷帅。
- 世界观主体是卡通人物、星舰、星港居民和轻科幻单位；不要把居民、活动图标或主要入口画成卡通动物。
- 界面整体颜色要鲜明，各单位、建筑、按钮、功能区要有清晰区分，不能糊成一片。
- 画面要清晰通透，避免低清、糊、像素格、马赛克格纹、过度锐化噪点或一格一格的块状质感。
- 不要冷峻硬核军事 HUD，不要纯扁平 UI，不要深色霓虹赛博风。
- 星舰是圆润但有机械结构的战机/载具，弱拟人，不要给星舰画脸。
- UI 可以有按钮、面板、货币条、血条等，并且直接生成 UI 文字。
- 顶部留出微信小游戏安全区，不要把关键控件放到最顶部。

## 出图时的执行口径

- 效果图要像正式游戏截图，不要做成海报、设定图或纯概念板。
- 角色、星舰、建筑、按钮、特效要在同一暖色基调下保持明显区分。
- 驾驶员负责承载人格和情绪，星舰负责承载载具感和火力感。
- 战斗画面可以更有火力和冲击，但仍要保持明亮温馨，不走暗黑硬核路线。
- UI 文字可以直接生成，但后续工程接入时仍以 Cocos 渲染文字为最终准绳。

## 立绘/单体透明底素材口径

- 星舰、驾驶员、敌人、图标、特效单体素材优先输出透明底 PNG；背景/场景源图不透明。
- 单体素材不要带复杂背景、地面、投影大底板、文字、水印、边框或展示 UI。
- 画面主体要完整，不裁头、不裁脚、不裁船翼，四周保留少量安全留边，方便工程缩放和裁切。
- 驾驶员立绘使用 3/4 视角，3-3.5 头身，表情克制、高冷酷帅但仍可爱；需要能从立绘裁出清晰头像。
- 驾驶员立绘不要拿武器/特效遮住脸，不要过度动态变形，不要婴儿脸或贴纸化。
- 我方星舰需要双套：3/4 展示立绘 + 俯视战斗 sprite。3/4 用于装配/船坞/收藏；俯视用于战斗。
- 同一艘星舰的 3/4 与俯视必须能看出是同一艘：主色、轮廓、核心结构、武器特征保持一致。
- 星舰 3/4 展示立绘要有金属机械结构、火力细节和能量点缀；圆润但仍是战机/载具，不给脸。
- 我方星舰俯视 sprite 以机头朝上为基准；敌人俯视 sprite 以机头/攻击方向朝下为基准。
- 敌人只需要俯视战斗 sprite，透明底；造型要一眼区分阵营和威胁，但仍是卡通化对手，不血腥压抑。
- 图标单体要在小尺寸下可读，轮廓清楚、颜色鲜明，优先保持同一套描边/高光/材质语言。
- 特效单体或关键帧要透明底，分清发射物、命中、爆炸/扩散、状态挂载四类用途。

## 禁区

- 禁止 2 头身婴儿脸、低龄贴纸感、过度卖萌。
- 禁止把星港居民、活动入口、功能图标的主体画成卡通动物或动物拟人。
- 禁止给星舰画表情、眼睛、嘴巴或明显人脸。
- 禁止把画面做成深色赛博霓虹、冷峻军事雷达 HUD、纯扁平线框 UI。
- 禁止让单位、按钮、建筑、背景颜色过于接近，导致功能区混在一起。
- 禁止在微信顶部安全区放关键按钮、货币、活动入口或战斗控制。
