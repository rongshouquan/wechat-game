## 0. 结论速览（TL;DR）
- **战斗引擎 + 全套基建可直接复用**（config/校验/存档+云mock/ads mock/analytics/防重/Node模拟器/vitest/微信build/灰盒UI）。总体**中等改造量，远小于重做**。
- 工作分三类：
  - **【架构偏差·必改】** ① 能量条 → 三类触发（§4.2）；② 插件 energy 槽名 / 形态对齐品质制。
  - **【需新增运行时】**（配置都齐了，**缺把配置翻译成战斗效果的"效果装配层"**）：驾驶员 / 星核 / 插件 / 基地养成。
  - **【战斗补 / 调参】** 敌方 5×7 可配置、普攻"原子炮"AoE、120s 超时确认。
- **关键利好（降低改造量）**：输入契约**已含** `pilotId/coreId/pluginIds` 槽位 + 归属/配置校验；引擎**已有** `coreEffectRef` 钩子；**AoE 动作积木已存在**（clear_barrage/line_pierce/backline_strike/burst_nuke）；43 张配置表 + 校验齐全。

## 2. 关键架构发现（亲验，已纠正盘点 3 处错误）
1. **敌方是 3×7、硬编码 const**——盘点速判表"5×7 已实现"**错误**。改 5×7 需同时改依赖常量的距离/排序逻辑（如 `ENEMY_COLS-1-u.col`，engine:704），并把行列**提到配置**（v1.0 要"可配置"）。
2. **超时配置驱动**（`timeLimitSec = enc.timeLimitSec`）——盘点"硬编码 30s"**不准**。120s 是配置数值问题，非架构。
3. **引擎用能量条触发大招**（`energy/ENERGY_FULL/passiveEnergyPerSec`，命中涨能、满能放 `ultimateEffectRef`）——与 v1.0「取消能量」冲突，是**最大一块改动**。动作积木(伤害/AoE/治疗/护盾/状态)可复用，**只换触发机制**。
4. **pilot/core/plugin 输入契约已就位**（`S7BattleInputSnapshot` 有槽位 + 归属/配置校验），但**只校验不接入战斗**——缺"效果装配层"。这是好消息：数据边界现成。
5. **引擎已有 `coreEffectRef` 钩子**（大招后触发一次 `core_trigger`）——星核有部分挂载点，但形态是"大招后触发"，与 v1.0 过载核心"替换普攻"不同，需扩展。

## 3. 分块改造顺序（建议·依赖驱动）

> 原则：**先地基后上层**；每块独立可跑、改完 `vitest` + 提交；每块都保持逻辑纯 TS。
> 每块标注：做什么 / 为何这个顺位 / 风险 / 可测产出。

- **块 0 · 战场对齐（低风险先行，立刻可测）**
  做什么：敌方 3×7 → **5×7 并提为配置**；确认/调超时到 **120s**。
  为何先：纯战斗参数、风险低、先把战场对齐 v1.0，给后续改造一个对的底。
  风险：改 const 牵动距离/排序/占格逻辑（engine:704/794/807），需连带改 + 回归测试。
  可测：现有战斗测试全绿 + 新增 5×7 占格/目标语义用例。

- **块 1 · 效果装配层（一切运行时的共同地基）** ★
  做什么：建"收集一艘船 星舰+驾驶员+插件+星核 的全部 effect → 喂给引擎结算"的装配层（v1.0 §4.6 积木系统）。
  为何先：驾驶员/星核/插件运行时都依赖它；输入契约已就位，正好往上接。
  风险：要定清 effect 积木分类（修正/触发/行为/动作）与叠加规则；需大量边界叠加测试。
  可测：给定一艘船的四层配置 → 装配出的 effect 列表/最终属性，纯函数可断言。

- **块 2 · 技能触发架构：能量条 → 三类触发** ★（最大改动）
  做什么：移除 energy 模型，改 **被动常驻 / 条件触发 / CD 型（开局即放第一次）**；复用现有 cast/动作结算。
  为何此位：触发机制是技能/星核/驾驶员共同依赖的"节拍器"，须在它们运行时之前定型。
  风险：最高。涉及引擎主循环步序重写（现 3/4/6 步是能量相关）、所有现有战斗测试基线会变。建议拆小步：先引入 CD 触发与能量并存 → 迁移 → 删能量。
  可测：三类触发各自时序用例；"不齐爆"回归。

- **块 3 · 星核运行时 + 过载核心（原子炮 AoE 普攻）**
  做什么：基于块1+块2 实现星核质变；落地新手核**过载核心**（普攻→原子炮：开局即放、以目标列为中心约半场 AoE、间隔 10s）。复用已有 AoE 动作 + 扩展 coreEffectRef。
  为何此位：依赖装配层与触发架构；过载核心是 v1.0 新手关键正反馈，优先级高。
  可测：装过载核心的船，开局一炮命中半场、10s 节流；普攻通道切换正确。

- **块 4 · 插件品质制 + 合成**
  做什么：插件接入装配层（定向数值修正）；实现品质（精良/优秀/传奇）+ 合成（3 同 → 1 高阶，槽内随机词条）+ 回收。槽名 `energy` → **技能(CD)槽**对齐 v1.0 §5.3。
  风险：合成/回收涉及存档与防重；槽名改动牵动配置与校验。

- **块 5 · 驾驶员运行时**
  做什么：驾驶能力（行为 AI：锁后排/护盾优先/嘲讽…）+ 驾驶天赋（独特被动），接入装配层；**不加原始属性**。
  为何靠后：依赖装配层 + 触发架构 + 目标语义（块0 的"最近/最远·第一/最后排"）。

- **块 6 · 基地养成 / 建筑运行时 + 货币对齐**
  做什么：`BuildingService` + 存档 buildings 字段；升级即时无等待、1-10 级；离线产出并入基地×星域系数（吸收采矿站）。货币按 §4 映射落地。
  风险：动存档结构 → 需迁移/重置说明。

- **块 7 · 活动：3 天行动 / 7 天扩张（去每日任务）**
  做什么：复用 DailyGoal 的"事件绑定 + 防重"做活动进度引擎；新增周期/里程碑/宝藏结算；去掉每日任务形态。
  为何最后：依赖前面产出的资源/星核/插件已能产出，活动才有东西可发。
