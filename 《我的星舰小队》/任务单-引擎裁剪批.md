# 任务单 · 引擎模块裁剪批（包体瘦身大杠杆 · Ron 在 Cocos 编辑器主导）

> 背景：主包最大头是引擎 `cocos-js/cc.js ≈ 2248KB`（占主包约 58%）。本项目是竖屏 2D Canvas 玩法（S7 灰盒 UI 全用 `Graphics` 手绘，无 3D/物理/骨骼/瓦片图/视频/富文本），当前引擎却编入了 26 个模块、含大量用不到的重模块。**裁掉这些能真实减重 ~640KB（不含 3d）**，是比删流程版脚本大得多的杠杆。
>
> **怎么操作**：Cocos Creator 编辑器 → 菜单 **项目 → 项目设置 → 功能裁剪（Feature Cropping）**→ 取消勾选下列模块 → 重新构建 wechatgame。编辑器会把结果写回 `game/settings/v2/packages/engine.json`。**不要手改 json**（依赖关系由编辑器自动解算，手改易漏依赖导致构建异常）。
>
> **每批一验一提交**：每批取消勾选后 → 重构建 → 用 `ls -la build/wechatgame/cocos-js/cc.js` 看字节数变化（核实真减重）→ 模拟器 + 真机跑一遍 S7（主线打一场 / 悬赏板 / 战斗 / 推演 / 回廊 / 趣事气泡）确认无异常 → 提交这一批。Claude 每批帮你核对字节数与冒烟结论。

## 绝对保留（勿动·关任一即白屏或功能坏）

`2d` · `base` · `ui` · `ui-skew` · `affine-transform` · `gfx-webgl` · `custom-pipeline` · `graphics`(灰盒UI主力·104处) · `tween`(UiFx.ts 在用) · `audio`(AudioSource 在用) · `mask`(收益低风险高·不动)

## 分批裁剪表（低风险先行·3d 压轴）

### 第 1 批（近零风险·一次性全关）— 估省 ~370KB
取消勾选：`physics-2d-box2d` · `tiled-map` · `rich-text` · `video` · `webview` · `websocket` · `intersection-2d` · `particle-2d`
- 依据：脚本层 0 处引用 RigidBody2D/Collider2D、TiledMap、RichText、VideoPlayer、WebView、WebSocket、intersection-2d、ParticleSystem2D；场景无任何相关组件。激励视频广告走 `wx.createRewardedVideoAd`（微信原生 API），与引擎 `video` 模块无关。
- 验证重点：整体启动 + 战斗 + 各 UI 页正常。

### 第 2 批 — 估省 ~165KB
取消勾选：`skeletal-animation` · `marionette` · `meshopt` · `primitive`
- 依据：无骨骼动画、无动画状态机图、无网格/3D 模型、无 3D 基础几何体。
- 验证重点：战斗演出 + UI 动效。

### 第 3 批（单独验）— 估省 ~40KB
取消勾选：`animation`
- 依据：脚本 0 处用 `cc.Animation`，动效走独立的 `tween`。
- ⚠️ 先确认 `tween` 仍勾选且动效正常再关；`animation` 在个别 3.8 版本可能被编辑器序列化隐性牵连，故单独一批验。

### 第 4 批（压轴·单独 commit·最严格验）— 乐观再省最多 ~300KB，风险最高
尝试取消勾选：`3d`（连带 Model/MeshRenderer/3D 光照/Skybox）
- ⚠️ **最大坑**：Cocos 3.8 的 2d/3d 拆分不彻底——即便全是 2D UI，关 3d 可能导致：场景里默认的 `DirectionalLight/Skybox/SceneGlobals` 反序列化报错，或 2D 渲染底层仍需 3d 的 gfx/render-scene 部分而**运行时白屏**。
- **落地规矩**：单独一轮、单独 commit、模拟器 + 真机都要严验；**关后一旦白屏或场景报错就回退保留 3d**（只留第 1–3 批成果 + 可单独关 meshopt/primitive 等 3d 周边）。300KB 是乐观上限，实际能省多少以关掉后真机不炸为准。

### 发布批（正式上线前）
取消勾选：`profiler`（性能探针面板，开发期保留、发布可关，省量小）

## 收益汇总（估算·以每批 cc.js 实际字节数为准）
- 第 1–3 批（稳拿）：**~575–640KB**
- 第 4 批 3d（能过才算）：叠加最多 **~300KB**
- 合计乐观上限：逼近 ~900KB+；保守承诺值：**~640KB**

> 提醒（呼应硬规则）：以上 KB 为 Cocos 3.8 模块经验区间，非逐模块精确测量；每批构建后对比 `cc.js` 字节数得真实值。"以为没用其实运行时用到"的模块（尤其 3d）已按要求宁标风险不轻言可删。
