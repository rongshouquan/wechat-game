## 目标（一句话）

拿着已校准的尺子，对全部 14 币种做"源→池→汇"第一性审视：每种资源从哪来、堆在哪、到哪去，找出**死水、溢出、断链、主导策略、体验隐患**，输出一份能直接指导第二子步精调顺序的审视报告。

## 审视范围与方法（硬规格）

1. **逐币种过一遍**（钱包 14 键，`RESOURCE_KEYS` 为准）：每币种列 源（哪些渠道、各占多少——用 `--income` 实测数据说话）/ 池（峰值库存、毕业结余）/ 汇（哪些消耗、够不够吃）。四档都看，普通档为主基准。
2. **初值表 §11 的 8 条线索逐条深挖**（星贝死水/非主力碎片沉淀/信标积压/打捞过度载重/星核通胀苗头/广告#1独大/毕业后星矿死币/三选一点名损耗）——每条给：实测证据（跑尺子拿数）→ 严重度（高/中/低）→ 至少两个候选解法 → 你的推荐与一句理由。
3. **8 条之外自由巡逻**：用第一性原理再扫一遍（每个系统玩家做完动作有没有正反馈？正反馈驱不驱动下一步？长期会不会枯竭/通胀/无意义？），发现新问题照 #2 格式补。
4. **体验红线对照**：每条建议标注它动没动 Ron 拍板的体验支柱（星核流速支柱 B1 §8 / 辅助战斗日耗时 5-10 分钟 / 广告只加速不卡死 / 零红点哲学 / 平滑不憋屈）——碰到支柱的必须显式说明。**2026-07-06 后新增支柱一并对照**：递进墙+7 天硬顶、肝党锚（≈日2.5h）、广告双轨（常规轨上限 25-30% / 黑市轨另立账本）——口径见 README 锁定决策与 GDD S13.6；审视时黑市轨当独立账本看。
5. **早期停玩惩罚专项**（总控验收时发现的盯点）：D10 附近停玩 2 天毕业晚 4 天、超出停玩本身 2 天，中后期则完全吸收——查根因（早期离线存储上限？巡逻未解锁？悬赏积压量小？），给候选解法。
6. **实证纪律**：每个结论必须带尺子实测数字或真源引文，禁止"感觉上"；跑尺子改参数做对照实验可以（那是审视手段），但**不留任何参数改动**（报告交付时 `git status` 干净、gate 942 绿）。
