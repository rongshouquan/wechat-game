《我的星舰小队》设计可视化（本地渲染版）

用途：把 v1.0 设计画成图，方便找漏洞/断点。本页用 mermaid.js 在浏览器里渲染，中文正常显示。若图没出来，多半是没联网取 CDN，可改用 VS Code 的 Mermaid 预览插件看 设计可视化-v0.1.md。

图① 主循环（玩家一天怎么转）

flowchart TD
  A["上线：看到基地变大、居民变多"] --> B["领离线积累 + 打捞带回的奖励"]
  B --> C["战前搭配：换 驾驶员/星舰/插件/星核 + 调站位"]
  C --> D["自动战斗·推主线关卡"]
  D --> E{"赢了吗？"}
  E -->|赢| F["拿软货币/信标/插件/碎片 + 首通解锁(本体/建筑/系统)"]
  E -->|卡关或输| C
  F --> G["回基地：升级/升阶/升星、扩建建筑"]
  G --> H["推进 3天行动 / 7天扩张、突破 Boss"]
  H --> I["开宝箱：星辉货舱 / 行动宝藏 / 扩张宝藏 → 星核等"]
  I --> J["小队更能打 + 基地更热闹"]
  J --> A
  D -. 战力不够 .-> K["卡关 → 渴望更强"]
  K -. 拿到牛星核 .-> L["兵种质变·画面更爽"]
  L -. 碾压推几关 .-> D

重点检查：失败后是否只回到「搭配/养成」而非「操作时机」；虚线「卡关→渴望→牛核→碾压→再卡」的爽点小循环有无断点。

图③ 四层战斗数据流（一个单位怎么组装出战斗力）

flowchart TD
  ship["星舰：基础属性 + 技能(普攻/被动/条件/CD) + 3插件槽+1星核槽<br/>『打什么·底子多强』"]
  pilot["驾驶员：驾驶能力(选目标/行为AI) + 驾驶天赋(独特被动)<br/>无原始属性 · 『怎么打·打谁』"]
  plugin["插件×3：定向数值·品质制<br/>『某方向补多少数值』"]
  core["星核×1：规则质变<br/>『发生什么质变』"]
  ship --> E["把四层的全部 effect 收集起来<br/>(修正/触发/行为/动作 四类积木)"]
  pilot --> E
  plugin --> E
  core --> E
  E --> ENG["战斗引擎：逐 tick 按规则结算"]
  ENG --> OUT["涌现出来的战斗表现(胜负 + 战报)"]
  subgraph EX["例：导弹舰"]
    e1["星舰给『齐射』动作"] --> e2["突袭手让齐射『专锁后排』"]
    e2 --> e3["武器插件加『暴击+溅射』数值"]
    e3 --> e4["小太阳星核把齐射变成『吸附爆炸的小太阳』"]
  end

重点检查：四层职责不重叠（星舰=动作本体、驾驶员=怎么用/打谁、插件=纯数值、星核=质变）；驾驶员是否真"不自带独立技能、不加原始属性"。

重点检查：每个建筑/系统是否都有「玩它的理由(产出)」和「它消耗啥」；解锁节奏 D1船坞→…→D7星核展厅 有无断层。
