Claude Code 开发原则（针对本项目）
1.阶段粒度控制
每个阶段的工作量必须设计为 ≤15分钟完成
阶段内只执行必要的功能，不做多余扩展
在以后每个新的一阶段开始前，你要先预估工作量，向我汇报该阶段大概的工作时间以及大概消耗的tokens，得到我的确认后才能开始。
2.输出限制
不在对话里生成不必要的代码或调试日志
阶段结束输出仅包含：
新增或修改的文件列表
关键功能点说明
已测功能确认
避免一次性生成大量代码块
3.自检与修复
每个阶段完成后必须自行真机/模拟器自测
发现问题优先自修复，最多尝试 3 次
仅在无法自修复时报告给你
4.项目规范
严格遵守微信小游戏规范
使用 Canvas 2D + CommonJS
触摸事件用 wx.onTouchStart/Move/End
尺寸用 screenWidth/screenHeight
禁止 DOM、window、document、shadowBlur 等不兼容 API
5.阶段交付内容
仅交付该阶段必须的文件和逻辑
不包含未来阶段内容
包含自测结果和已知问题说明
6.Token 与思考控制
每个阶段执行过程中，避免生成超长代码和冗余信息
不允许“长时间 thinking”或“almost done thinking”导致阻塞
